# Agent Instructions for Building Projects with Derp

This document provides step-by-step instructions for an AI agent developing a project that uses the `derp` package — an async Python backend framework for PostgreSQL with built-in auth, storage, payments, and a Drizzle-inspired migration system.

## Prerequisites

- Python 3.12+
- PostgreSQL database
- `uv` package manager (recommended)

## Step 1: Project Setup

### Initialize the project

```bash
uv init my-project
cd my-project
uv add derp
```

### Create the configuration file

Run `derp init` or create `derp.toml` manually in the project root:

```toml
[database]
db_url = "$DATABASE_URL"         # Environment variable, NOT the actual URL
schema_path = "src/schema.py"    # Path to your schema file(s)
migrations_dir = "migrations"
```

**Important**: Database URLs are NEVER stored directly in config. Use `$ENV_VAR` syntax to reference environment variables. Create a `.env` file for local development:

```
DATABASE_URL=postgresql://user:pass@localhost:5432/mydb
```

### Recommended project structure

```
my-project/
├── derp.toml
├── .env
├── pyproject.toml
├── src/
│   ├── schema.py          # Table definitions
│   ├── main.py            # Application entry point
│   ├── dependencies.py    # FastAPI dependencies (if using FastAPI)
│   ├── schemas.py         # Pydantic request/response models
│   └── routers/           # API route handlers
│       ├── __init__.py
│       └── ...
└── migrations/            # Generated by `derp generate`
```

Alternatively, you can use `schema_path = "src/*"` to load tables from all Python files in a directory.

## Step 2: Define Tables

### Basic table definition

Tables inherit from `Table` and use `Field()` for column definitions:

```python
from __future__ import annotations

import uuid
from datetime import datetime

from derp.orm import Field, Table
from derp.orm.fields import (
    UUID,
    Serial,
    Varchar,
    Text,
    Integer,
    Boolean,
    Timestamp,
    ForeignKey,
    ForeignKeyAction,
)


class User(Table, table="users"):
    id: int = Field(Serial(), primary_key=True)
    name: str = Field(Varchar(255))
    email: str = Field(Varchar(255), unique=True)
    age: int | None = Field(Integer(), nullable=True)
    is_active: bool = Field(Boolean(), default="true")
    created_at: datetime = Field(Timestamp(with_timezone=True), default="now()")


class Post(Table, table="posts"):
    id: uuid.UUID = Field(UUID(), primary_key=True, default="gen_random_uuid()")
    author_id: int = Field(
        Integer(),
        foreign_key=ForeignKey(User, on_delete=ForeignKeyAction.CASCADE),
        index=True,
    )
    title: str = Field(Varchar(255))
    content: str = Field(Text())
    published: bool = Field(Boolean(), default="false")
    created_at: datetime = Field(Timestamp(with_timezone=True), default="now()")
```

### Key rules for table definitions

1. **Always use `from __future__ import annotations`** in all files.
2. **Fields default to NOT NULL**. Use `nullable=True` for optional columns.
3. **Use `Table.c.column`** (not `Table.column`) when building queries. The `.c` accessor returns `FieldInfo` objects needed by the query builder.
4. **Foreign keys reference table classes directly**: `ForeignKey(User)` or use a string like `ForeignKey("users.id")`.
5. **Default values are SQL expressions** passed as strings: `default="now()"`, `default="gen_random_uuid()"`, `default="true"`.

### Available field types

| Type | Python Type | Description |
|------|-------------|-------------|
| `Serial()` | `int` | Auto-incrementing 4-byte integer |
| `BigSerial()` | `int` | Auto-incrementing 8-byte integer |
| `SmallInt()` | `int` | 2-byte integer |
| `Integer()` | `int` | 4-byte integer |
| `BigInt()` | `int` | 8-byte integer |
| `Varchar(n)` | `str` | Variable-length string with limit |
| `Char(n)` | `str` | Fixed-length string |
| `Text()` | `str` | Unlimited length string |
| `Boolean()` | `bool` | Boolean |
| `Timestamp(with_timezone=False)` | `datetime` | Timestamp |
| `Date()` | `date` | Date |
| `Time(with_timezone=False)` | `time` | Time |
| `Interval()` | `timedelta` | Time interval |
| `Numeric(precision, scale)` | `Decimal` | Exact numeric |
| `Real()` | `float` | 4-byte float |
| `DoublePrecision()` | `float` | 8-byte float |
| `UUID()` | `uuid.UUID` | UUID |
| `JSON()` | `Any` | JSON |
| `JSONB()` | `Any` | Binary JSON (prefer over JSON) |
| `Array(element_type)` | `list` | Array of another type |
| `Vector(dim)` | `list[float]` | Vector type (for embeddings) |
| `TSVector(language)` | `str` | Full-text search vector |
| `Enum(enum_class)` | `StrEnum` | PostgreSQL enum |

### Field options

```python
Field(
    field_type,           # Required: Serial(), Varchar(255), etc.
    primary_key=False,    # Mark as primary key
    unique=False,         # UNIQUE constraint
    nullable=False,       # Allow NULL (default: NOT NULL)
    default=None,         # SQL default expression as string
    foreign_key=None,     # ForeignKey reference
    index=False,          # Create index on column
)
```

### Foreign key actions

`ForeignKeyAction.CASCADE`, `SET_NULL`, `SET_DEFAULT`, `RESTRICT`, `NO_ACTION`

## Step 3: Run Migrations

After defining tables, generate and apply migrations:

```bash
# Generate a migration from your schema changes
derp generate --name initial

# Review the generated SQL in migrations/0000_initial/migration.sql

# Apply the migration to the database
derp migrate
```

For development, you can push schema directly (skips migration files):

```bash
derp push
```

Other useful CLI commands:

```bash
derp status       # Check migration status
derp check        # Verify schema matches database
derp pull         # Pull schema from existing database
derp drop         # Drop all tables (development only)
derp studio       # Launch database GUI
```

### Custom migrations

For operations the differ can't auto-detect (data migrations, custom SQL):

```bash
derp generate --name seed_data --custom
# Edit the generated migration.sql manually
derp migrate
```

## Step 4: Query Data

### Connect to the database

Use `DerpClient` as the main entry point:

```python
from derp import DerpClient, DerpConfig

config = DerpConfig.load()
client = DerpClient(config)

async with client:
    users = await client.db.select(User).execute()
```

Or use `DatabaseEngine` directly for database-only usage:

```python
from derp.orm import DatabaseEngine

db = DatabaseEngine("postgresql://user:pass@localhost:5432/mydb")

async with db:
    users = await db.select(User).execute()
```

### SELECT queries

```python
# Select all rows — returns list[User]
users = await db.select(User).execute()

# Select specific columns — returns list[dict[str, Any]]
results = await db.select(User.c.id, User.c.name).execute()

# WHERE clause
users = await db.select(User).where(User.c.age > 18).execute()

# Complex WHERE with AND/OR
users = await db.select(User).where(
    ((User.c.name == "Alice") | (User.c.name == "Bob"))
    & (User.c.age > 18)
).execute()

# ORDER BY, LIMIT, OFFSET
users = await (
    db.select(User)
    .order_by(User.c.created_at, asc=False)
    .limit(10)
    .offset(20)
    .execute()
)

# First result (returns User | None)
user = await db.select(User).where(User.c.id == 1).first_or_none()

# First result (raises if not found)
user = await db.select(User).where(User.c.id == 1).first()
```

### Comparison operators

All operators work on `Table.c.column` (FieldInfo objects):

```python
User.c.id == 1                     # =
User.c.id != 1                     # !=
User.c.age > 18                    # >
User.c.age >= 18                   # >=
User.c.age < 65                    # <
User.c.age <= 65                   # <=
User.c.id.in_([1, 2, 3])          # IN
User.c.id.not_in([4, 5, 6])       # NOT IN
User.c.name.like("%Alice%")        # LIKE
User.c.name.ilike("%alice%")       # ILIKE (case-insensitive)
User.c.age.is_null()               # IS NULL
User.c.age.is_not_null()           # IS NOT NULL
User.c.age.between(18, 65)         # BETWEEN

# Logical operators
(User.c.age > 18) & (User.c.age < 65)      # AND
(User.c.name == "Alice") | (User.c.name == "Bob")  # OR
~(User.c.age > 18)                          # NOT
```

### INSERT queries

```python
# Insert without returning
await db.insert(User).values(name="Bob", email="bob@example.com").execute()

# Insert with RETURNING full model — returns User
user = await (
    db.insert(User)
    .values(name="Bob", email="bob@example.com")
    .returning(User)
    .execute()
)

# Insert with RETURNING specific columns — returns dict
result = await (
    db.insert(User)
    .values(name="Bob", email="bob@example.com")
    .returning(User.c.id, User.c.name)
    .execute()
)
```

### UPDATE queries

```python
# Update without returning
await db.update(User).set(name="Robert").where(User.c.id == 1).execute()

# Update with RETURNING — returns list[User]
users = await (
    db.update(User)
    .set(name="Robert", age=30)
    .where(User.c.age > 18)
    .returning(User)
    .execute()
)
```

### DELETE queries

```python
# Delete without returning
await db.delete(User).where(User.c.id == 1).execute()

# Delete with RETURNING — returns list[User]
deleted = await (
    db.delete(User)
    .where(User.c.is_active == False)
    .returning(User)
    .execute()
)
```

### JOINs

```python
# INNER JOIN
results = await (
    db.select(User.c.name, Post.c.title)
    .from_(User)
    .inner_join(Post, User.c.id == Post.c.author_id)
    .where(Post.c.published == True)
    .execute()
)

# Other join types
.left_join(table, condition)
.right_join(table, condition)
.full_join(table, condition)
.cross_join(table, condition)
```

### Transactions

```python
async with db.transaction():
    user = await (
        db.insert(User)
        .values(name="Alice", email="alice@example.com")
        .returning(User)
        .execute()
    )
    await (
        db.insert(Post)
        .values(author_id=user.id, title="Hello", content="World")
        .execute()
    )
    # Commits on success, rolls back on exception
```

### Raw SQL

For queries the builder doesn't support:

```python
result = await db.execute(
    "SELECT COUNT(*) as count FROM users WHERE age > $1",
    [18],
)
count = result[0]["count"]
```

## Step 5: Integrate with FastAPI

### Application lifecycle

```python
from __future__ import annotations

from collections.abc import AsyncIterator
from contextlib import asynccontextmanager

from dotenv import load_dotenv
from fastapi import FastAPI

from derp import DerpClient, DerpConfig

load_dotenv()


@asynccontextmanager
async def lifespan(app: FastAPI) -> AsyncIterator[None]:
    config = DerpConfig.load()
    client = DerpClient(config)
    await client.connect()
    app.state.derp_client = client
    yield
    await client.disconnect()


app = FastAPI(lifespan=lifespan)
```

### Dependencies

```python
from fastapi import Depends, Request
from derp import DerpClient


def get_derp(request: Request) -> DerpClient:
    return request.app.state.derp_client
```

### Route handlers

```python
from fastapi import APIRouter, Depends, HTTPException, status

router = APIRouter(prefix="/posts", tags=["posts"])


@router.get("")
async def list_posts(derp: DerpClient = Depends(get_derp)):
    return await (
        derp.db.select(Post)
        .where(Post.c.published == True)
        .order_by(Post.c.created_at, asc=False)
        .execute()
    )


@router.post("", status_code=status.HTTP_201_CREATED)
async def create_post(title: str, content: str, derp: DerpClient = Depends(get_derp)):
    return await (
        derp.db.insert(Post)
        .values(title=title, content=content, author_id=current_user.id)
        .returning(Post)
        .execute()
    )
```

## Step 6: Add Authentication (Optional)

### Configure auth in `derp.toml`

```toml
[email]
site_name = "My App"
site_url = "https://example.com"
from_email = "noreply@example.com"
smtp_host = "smtp.example.com"
smtp_port = 587
smtp_user = "$SMTP_USER"
smtp_password = "$SMTP_PASSWORD"

[auth]
enable_signup = true
enable_confirmation = false

[auth.jwt]
secret = "$JWT_SECRET"
access_token_expire_minutes = 15
refresh_token_expire_days = 7
```

**Important**: Auth requires email to be configured. The framework will raise an error if `[auth]` is set without `[email]`.

### Use auth in routes

```python
from derp.auth.exceptions import (
    InvalidCredentialsError,
    UserAlreadyExistsError,
    PasswordValidationError,
)

@router.post("/signup")
async def signup(email: str, password: str, derp: DerpClient[User] = Depends(get_derp)):
    try:
        user, tokens = await derp.auth.sign_up(
            email=email,
            password=password,
            user_agent=request.headers.get("User-Agent"),
            ip_address=request.client.host,
        )
    except UserAlreadyExistsError:
        raise HTTPException(status_code=409, detail="User already exists")
    except PasswordValidationError as e:
        raise HTTPException(status_code=400, detail=e.message)

    return {
        "user": user,
        "access_token": tokens.access_token,
        "refresh_token": tokens.refresh_token,
    }

@router.post("/signin")
async def signin(email: str, password: str, derp: DerpClient[User] = Depends(get_derp)):
    user, tokens = await derp.auth.sign_in_with_password(
        email=email,
        password=password,
        user_agent=request.headers.get("User-Agent"),
        ip_address=request.client.host,
    )
    return {"access_token": tokens.access_token, "refresh_token": tokens.refresh_token}
```

### JWT token validation dependency

```python
from derp.auth.jwt import decode_token
from derp.auth.exceptions import InvalidTokenError, TokenExpiredError, SessionExpiredError

async def get_current_user(
    request: Request, derp: DerpClient[User] = Depends(get_derp)
) -> User:
    auth_header = request.headers.get("Authorization")
    if not auth_header or not auth_header.startswith("Bearer "):
        raise HTTPException(status_code=401, detail="Missing token")

    token = auth_header.split(" ")[1]

    try:
        payload = decode_token(derp.auth._config.jwt, token)
        await derp.auth.validate_session(token)
        user = await derp.auth.get_user(payload.sub)
        if not user:
            raise HTTPException(status_code=401, detail="User not found")
        request.state.session_id = payload.session_id
        return user
    except (TokenExpiredError, InvalidTokenError, SessionExpiredError):
        raise HTTPException(status_code=401, detail="Invalid or expired token")
```

## Common Patterns and Gotchas

### Always use `.c` for queries

```python
# CORRECT
users = await db.select(User).where(User.c.id == 1).execute()

# WRONG — will not work
users = await db.select(User).where(User.id == 1).execute()
```

### UUID columns must be compared as strings

When comparing UUID columns with Python UUID values, cast to string:

```python
conversation = await (
    db.select(Conversation)
    .where(Conversation.c.id == str(conversation_id))
    .first_or_none()
)
```

### No automatic relationships

Derp does not have ORM-style lazy-loading relationships. Fetch related data explicitly:

```python
# Get posts for a user
posts = await db.select(Post).where(Post.c.author_id == user.id).execute()

# Or use a JOIN for combined data
results = await (
    db.select(User.c.name, Post.c.title)
    .from_(User)
    .inner_join(Post, User.c.id == Post.c.author_id)
    .execute()
)
```

### Use transactions for multi-step operations

```python
async with db.transaction():
    # All queries here are atomic
    await db.insert(Order).values(...).execute()
    await db.update(Inventory).set(stock=new_stock).where(...).execute()
```

### Prefer JSONB over JSON

`JSONB` is more efficient for queries and supports indexing. Use `JSON` only when you need to preserve key order.

### Use `enum.StrEnum` for string constants

```python
import enum

class Status(enum.StrEnum):
    ACTIVE = "active"
    INACTIVE = "inactive"
    SUSPENDED = "suspended"
```

### Cursor-based pagination

```python
query = (
    db.select(Post)
    .order_by(Post.c.created_at, asc=False)
    .limit(limit)
)

if cursor_id:
    cursor = await db.select(Post).where(Post.c.id == str(cursor_id)).first_or_none()
    if cursor:
        query = query.where(Post.c.created_at < cursor.created_at)

posts = await query.execute()
```
